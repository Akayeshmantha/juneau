{todo} @RemoteMethod Annotation

What if you want fine-tuned control over which methods are exposed in an interface instead of just all public methods?
For this, the {@link oajrc.remote.RemoteMethod @RemoteMethod} annotation is provided.
It can be applied to individual interface methods to only expose those methods through the remote interface servlet.

For example, to expose only the first 2 methods in our IAddressBook interface...

public interface IAddressBook { @RemoteMethod Person createPerson(CreatePerson cp) throws Exception; @RemoteMethod Person findPerson(int id); Address findAddress(int id); Person findPersonWithAddress(int id); }
