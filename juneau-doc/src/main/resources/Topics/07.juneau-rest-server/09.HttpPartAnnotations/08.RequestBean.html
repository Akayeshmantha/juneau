{new} @RequestBean

The {@link org.apache.juneau.http.annotation.RequestBean @RequestBean} annotation can be applied to a parameter or parameter type of a @RestMethod-annotated method to identify it as an interface for retrieving HTTP parts through a bean interface.

Example:

@RestMethod(path="/mypath/{p1}/{p2}/*") public void myMethod(@RequestBean MyRequestBean rb) {...} public interface MyRequestBean { @Path // Path variable name inferred from getter. String getP1(); @Path("p2") String getX(); @Path("/*") String getRemainder(); @Query String getQ1(); // Schema-based query parameter: Pipe-delimited lists of comma-delimited lists of integers. @Query( collectionFormat="pipes" items=@Items( items=@SubItems( collectionFormat="csv" type="integer" ) ) ) int[][] getQ3(); @Header("*") Map<String,Object> getHeaders();

// Same as above but annotation defined on interface. @RestMethod(path="/mypath/{p1}/{p2}/*") public void myMethod(MyRequestBean rb) {...} @RequestBean public interface MyRequestBean {...}

The return types of the getters must be the supported parameter types for the HTTP-part annotation used.
Schema-based serialization and parsing is used just as if used as individual parameter types.
