{new} @ResponseStatuses

The {@link org.apache.juneau.http.annotation.ResponseStatuses @ResponseStatuses} annotation is used to associate multiple {@link org.apache.juneau.http.annotation.ResponseStatus @ResponseStatus} annotations to the same parameter or class.

Example:

@RestMethod(name="GET", path="/user/login") public void login( @FormData("username") String username, @FormData("password") String password, @ResponseStatuses={ @ResponseStatus(200) @ResponseStatus(code=401, description="Invalid user/pw") } Value<Integer> status ) { if (! isValid(username, password)) status.set(401); else status.set(200); }

The other option is to apply this annotation to a subclass of {@link org.apache.juneau.Value} which often leads to a cleaner REST method:

@ResponseStatuses={ @ResponseStatus(200) @ResponseStatus(code=401, description="Invalid user/pw") } public class LoginStatus extends Value<Integer> {} @RestMethod(name="GET", path="/user/login") public void login( @FormData("username") String username, @FormData("password") String password, LoginStatus status ) { if (! isValid(username, password)) status.set(401); else status.set(200); }

The attributes on this annotation are used to populate the generated Swagger for the method.
In this case, the Swagger is populated with the following:

'/user/login': { get: { responses: { 200: { description: 'OK' }, 401: { description: 'Invalid user/pw' } } } }
