The {@link org.apache.juneau.http.annotation.ResponseBody @ResponseBody} annotation is used to identify a placeholder parameter for an HTTP response.

  • {@link org.apache.juneau.http.annotation.ResponseBody ResponseBody}
    • {@link org.apache.juneau.http.annotation.ResponseBody#api() api()} - Free-form Swagger JSON.
    • {@link org.apache.juneau.http.annotation.ResponseBody#example() example()} - Serialized example.
    • {@link org.apache.juneau.http.annotation.ResponseBody#examples() examples()} - Serialized examples per media type.
    • {@link org.apache.juneau.http.annotation.ResponseBody#partSerializer() partSerializer()} - Override the part serializer.
    • {@link org.apache.juneau.http.annotation.ResponseBody#schema() schema()} - Swagger schema.
    • {@link org.apache.juneau.http.annotation.ResponseBody#usePartSerializer() usePartSerializer()} - Use the HTTP-Part serializer for serializing the body.

It can be used in the following locations:

  • Parameters on @RestMethod-annotated methods.
  • Classes of parameters on @RestMethod-annotated methods.
  • @RestMethod-annotated methods.
Parameters on @RestMethod-annotated methods

When used on a method parameter, this annotation is used to identify a placeholder {@link org.apache.juneau.Value} object that can be used to return the body of the response.

@RestMethod(path="/pet/{petId}") public void getPet(@Path("petId") int id, @ResponseBody Value<Pet> body) { body.set(getPet(id)); }

The behavior is identical to the following:

@RestMethod(path="/pet/{petId}") public Pet getPet(@Path("petId") int id) { return getPet(id); }

You may ask why you would use ever use this. The answer is that it's possible to have your method returns something else such as an HTTP response status or header value. This annotation allows you to set a response body when your method is returning one of these.
Also, the annotation allows you to define swagger validation and documentation for your body.

Classes of parameters on @RestMethod-annotated methods

The @ResponseBody annotation can also be applied to types.

@RestMethod(path="/pet/{petId}") public void getPet(@Path("petId") int id, Value<Pet> body) { body.set(getPet(id)); }

@ResponseBody( schema=@Schema( description="Pet bean" ) ) public class Pet {...}

Note that this can often be cleaner-looking than usage on the Java method parameter.
