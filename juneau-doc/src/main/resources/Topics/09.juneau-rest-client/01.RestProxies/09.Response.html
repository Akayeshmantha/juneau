{new} @Response

The {@link oaj.http.annotation.Response @Response} annotation can be applied to either @RemoteMethod-annotated methods or types returned by @RemoteMethod-annotated methods.

  • {@link oaj.http.annotation.Response Response}
    • {@link oaj.http.annotation.Response#partParser() partParser} - Override the part parser.
    • {@link oaj.http.annotation.Response#schema() schema} - Swagger schema.
    • {@link oaj.http.annotation.Response#usePartParser() usePartParser} - Use the HTTP-Part parser for parsing the response body.
On @RemoteMethod-annotated methods

As shown previously, the @Response annotation can be added to your @RemoteMethod-annotated method directly to specify whether the response body should use the OpenAPI part parser.

@RemoteResource public interface PetStore { // Force the part parser to be used for the response body. @RemoteMethod @Response(usePartParser=true) String postPet(...); }

It can also be used to provide schema information about the response body to provide OpenAPI schema based part parsing and validation.

Response interfaces

The @Response annotation can also be used to define interfaces for retrieving response parts using a bean-like proxies.

@RemoteResource public interface PetStore { @RemoteMethod CreatePetResponse postPet(...); }

@Response public interface CreatePetResponse { @ResponseBody Pet getBody(); @ResponseHeader("E-Tag") UUID getUUID(); @ResponseStatus int getStatus(); }

PetStore store = restClient.getRemoteResource(PetStore.class, "http://localhost:10000"); CreatePetResponse response = store.postPet(...); Pet pet = response.getBody(); UUID uuid = response.getUUID(); int status = response.getStatus();

The annotated methods must be no-arg. They can be named anything.

Any of the following annotations can be used on the methods:

  • {@link oaj.http.annotation.ResponseBody ResponseBody}
  • {@link oaj.http.annotation.ResponseHeader ResponseHeader}
  • {@link oaj.http.annotation.ResponseStatus ResponseStatus}
