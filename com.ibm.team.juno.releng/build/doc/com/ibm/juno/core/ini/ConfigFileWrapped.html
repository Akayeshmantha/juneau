Class ConfigFileWrapped

  • java.lang.Object
    • com.ibm.juno.core.ini.ConfigFile
      • com.ibm.juno.core.ini.ConfigFileWrapped
  • All Implemented Interfaces:
    Map<String,Section>

    public final class ConfigFileWrapped
extends ConfigFile
    Wraps an instance of ConfigFileImpl in an interface that will automatically replace StringVarResolver variables.

    The ConfigFile.getResolving(StringVarResolver) returns an instance of this class.

    This class overrides the ConfigFile.getString(String, String) to resolve string variables. All other method calls are passed through to the inner config file.

    Author:
    James Bognar (jbognar@us.ibm.com)

        public ConfigFile addHeaderComments(String section,
                           String... headerComments)
        Description copied from class: ConfigFile
        Adds header comments to the specified section.

        Header comments are defined as lines that start with "#" immediately preceding a section header "[section]". These are handled as part of the section itself instead of being interpreted as comments in the previous section.

        Header comments can be of the following formats...

        • "# comment" - A comment line.
        • "comment" - Anything else (will automatically be prefixed with "# ").

        If the section does not exist, it will automatically be created.
