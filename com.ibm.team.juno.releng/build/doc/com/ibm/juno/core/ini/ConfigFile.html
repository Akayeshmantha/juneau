        • "# comment" - A comment line.

        If the section does not exist, it will automatically be created.

      • addHeaderComments

        public abstract ConfigFile addHeaderComments(String section,
                           String... headerComments)
        Adds header comments to the specified section.

        Header comments are defined as lines that start with "#" immediately preceding a section header "[section]". These are handled as part of the section itself instead of being interpreted as comments in the previous section.

        Header comments can be of the following formats...

        • "# comment" - A comment line.
        • "comment" - Anything else (will automatically be prefixed with "# ").

        If the section does not exist, it will automatically be created.

        Parameters:
        section - The name of the section to add lines to, or null to add to the default section.
        headerComments - The comment lines to add to the section.
        Returns:
        This object (for method chaining).
        Throws:
        UnsupportedOperationException - If config file is read only.

      • getObject

        public final <T> T getObject(Class<T> c,
              String key)
                  throws ParseException
        Gets the entry with the specified key and converts it to the specified value.

        The key can be in one of the following formats...

        • "key" - A value in the default section (i.e. defined above any [section] header).
        • "section/key" - A value from the specified section.

        If the class type is an array, the value is split on commas and converted individually.

        If you specify a primitive element type using this method (e.g. int.class, you will get an array of wrapped objects (e.g. Integer[].class.

        Parameters:
        c - The class to convert the value to.
        key - The key. See getString(String) for a description of the key.
        Returns:
        The value, or null if the section or key does not exist.
        Throws:
        ParseException - If parser could not parse the value or if a parser is not registered with this config file.

      • put

        public final String put(String key,
         Object value,
         boolean encoded)
                 throws SerializeException
        Adds or replaces an entry with the specified key with the specified value.

        The format of the entry depends on the data type of the value.

        • Simple types (String, Number, Boolean, primitives) are serialized as plain strings.
        • Arrays and collections of simple types are serialized as comma-delimited lists of plain strings.
        • Other types (e.g. beans) are serialized using the serializer registered with this config file.
        • Arrays and collections of other types are serialized as comma-delimited lists of serialized strings of each entry.
        Parameters:
        key - The key. See getString(String) for a description of the key.
        value - The new value.
        encoded - If true, value is encoded by the registered encoder when the config file is persisted to disk.
        Returns:
        The previous value, or null if the section or key did not previously exist.
        Throws:
        SerializeException - If serializer could not serialize the value or if a serializer is not registered with this config file.
        UnsupportedOperationException - If config file is read only.

        public abstract ConfigFile getResolving(StringVarResolver vr,

        Returns a wrapped instance of this config file where calls to getters have their values first resolved by the specified StringVarResolver.

        Parameters:
        vr - The variable resolver for resolving string variables.
        augment - - If false, the specified var resolver replaces the existing var resolver. If true, then if the config file has an associated var resolver with it, that var resolver will be added as a parent to the specified var resolver.
