    Specialized content handler for correctly handling XML element content based on the type attribute of the element.

    If the type attribute is "xhtml" the content is treated as XML. Otherwise, it's treated as plain text.

        Reads XML element content the specified reader and sets the appropriate value on the specified bean.

        When this method is called, the attributes have already been parsed and set on the bean. Therefore, if the content handling is different based on some XML attribute (e.g. type="text/xml" vs type="text/plain") then that attribute value can be obtained via the set bean property.

        Specified by:
        parse in interface XmlContentHandler<Text>
        Parameters:
        r - The XML stream reader. When called, the reader is positioned on the element containing the text to read. For example, calling r.getElementText() can be called immediately to return the element text if the element contains only characters and whitespace. However typically, the stream is going to contain XML elements that need to be handled special (otherwise you wouldn't need to use an XmlContentHandler to begin with).
        text - The bean where the parsed contents are going to be placed. Subclasses determine how the content maps to values in the bean. However, typically the contents map to a single property on the bean.
