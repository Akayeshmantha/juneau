Class Lockable

  • java.lang.Object
    • com.ibm.juno.core.Lockable
  • All Implemented Interfaces:
    Cloneable
    Direct Known Subclasses:
    BeanContextFactory, CoreApi, ParserGroup, SerializerGroup

    public abstract class Lockable
extends Object
implements Cloneable
    Superclass of all classes that have a locked state.

    Used to mark bean contexts, serializers, and parsers as read-only so that settings can no longer be modified.

    Also keeps track of when the object has been cloned and allows for lazy cloning through the onUnclone() method. The idea behind this is that certain expensive fields don't need to be cloned unless the object is actually being modified.

    Calling lock() on the object causes it to be put into a read-only state. Once called, subsequent calls to checkLock() will cause LockedExceptions to be thrown.

    As a rule, cloned objects are unlocked by default.
