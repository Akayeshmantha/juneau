Interface XmlContentHandler<T>

  • Type Parameters:
    T - The class type of the bean
    All Known Subinterfaces:
    XmlContentHandler.NULL
    All Known Implementing Classes:

    public interface XmlContentHandler<T>
    Customization class that allows a bean (or parts of a bean) to be serialized as XML text or mixed content.

    For example, the ATOM specification allows text elements (e.g. title, subtitle...) to be either plain text or XML depending on the value of a type attribute. The behavior of text escaping thus depends on that attribute.

    <feed xmlns="http://www.w3.org/2005/Atom"> <title type="html"> &lt;p&gt;&lt;i&gt;This is the title&lt;/i&gt;&lt;/p&gt; </title> <title type="xhtml"> <div xmlns="http://www.w3.org/1999/xhtml"> <p><i>This is the subtitle</i></p> </div> </title> </feed>

    The ATOM Text class (the implementation for both the <title> and <subtitle> tags shown above) then associates a content handler through the Xml.contentHandler() annotation on the bean property containing the text, like so...

    @Xml(format=ATTR) public String getType() { return type; } @Xml(format=CONTENT, contentHandler=TextContentHandler.class) public String getText() { return text; } public void setText(String text) { this.text = text; }

    The content handler that transforms the output is shown below...

    public static class TextContentHandler implements XmlContentHandler<Text> { @Override public void parse(XMLStreamReader r, Text text) throws Exception { String type = text.type; if (type != null && type.equals("xhtml")) text.text = decode(readXmlContents(r).trim()); else text.text = decode(r.getElementText().trim()); } @Override public void serialize(XmlSerializerWriter w, Text text) throws Exception { String type = text.type; String content = text.text; if (type != null && type.equals("xhtml")) w.encodeTextInvalidChars(content); else w.encodeText(content); } }

    Notes
    • The Xml.contentHandler() annotation can only be specified on a bean class, or a bean property of format XmlFormat.CONTENT.

      • parse

        void parse(XMLStreamReader r,
         T bean)
           throws Exception
        Reads XML element content the specified reader and sets the appropriate value on the specified bean.

        When this method is called, the attributes have already been parsed and set on the bean. Therefore, if the content handling is different based on some XML attribute (e.g. type="text/xml" vs type="text/plain") then that attribute value can be obtained via the set bean property.

        Parameters:
        r - The XML stream reader. When called, the reader is positioned on the element containing the text to read. For example, calling r.getElementText() can be called immediately to return the element text if the element contains only characters and whitespace. However typically, the stream is going to contain XML elements that need to be handled special (otherwise you wouldn't need to use an XmlContentHandler to begin with).
        bean - The bean where the parsed contents are going to be placed. Subclasses determine how the content maps to values in the bean. However, typically the contents map to a single property on the bean.
        Throws:
        Exception - If any problem occurs. Causes parse to fail.

      • serialize

        void serialize(XmlSerializerWriter w,
             T bean)
               throws Exception
        Writes XML element content from values in the specified bean.
        Parameters:
        w - The XML output writer. When called, the XML element/attributes and whitespace/indentation (if enabled) have already been written to the stream. Subclasses must simply write the contents of the element.
        bean - The bean whose values will be converted to XML content.
        Throws:
        Exception - If any problems occur. Causes serialize to fail.
