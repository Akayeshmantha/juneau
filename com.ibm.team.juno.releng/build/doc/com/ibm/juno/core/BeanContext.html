com.ibm.juno.core

Class BeanContext

  • java.lang.Object
    • com.ibm.juno.core.BeanContext

  • public final class BeanContext
extends Object
    Core class of the Juno architecture.

    This class servers multiple purposes:

    • Provides the ability to wrap beans inside Map interfaces.
    • Serves as a repository for metadata on POJOs, such as associated filters, property namers, etc... which are used to tailor how POJOs are serialized and parsed.
    • Serves as a common utility class for all Serializers and Parsers for serializing and parsing Java beans.
    Bean Contexts

    Typically, it will be sufficient to use the existing DEFAULT contexts for creating bean maps. However, if you want to tweak any of the settings on the context, you must either clone the default context or create a new one from scratch (whichever is simpler for you). You'll notice that this context class uses a fluent interface for defining settings.

    Bean context factories can be locked using the BeanContextFactory.lock() method. This makes the context settings read-only. Attempting to change a setting on a locked context will cause a LockedException to be thrown. The default context is locked by default. Cloning a locked context using the Object.clone() method produces a new unlocked context. Locking a context is optional, although it can prevent errors where bean contexts, parsers, or serializers start behaving differently because a setting was changed. As a general rule, if you want to change a setting on an existing context, you should clone it, modify the setting(s), and then lock the new context.

    BeanContext settings
    BeanContexts have several settings that can be used to tweak behavior on how beans are handled.

    Some settings (e.g. BeanContextProperties.BEAN_beansRequireDefaultConstructor) are used to differentiate between bean and non-bean classes. Attempting to create a bean map around one of these objects will throw a BeanRuntimeException. The purpose for this behavior is so that the serializers can identify these non-bean classes and convert them to plain strings using the Object.toString() method.

    Some settings (e.g. BeanContextProperties.BEAN_beanFieldVisibility) are used to determine what kinds of properties are detected on beans.

    Some settings (e.g. BeanContextProperties.BEAN_beanMapPutReturnsOldValue) change the runtime behavior of bean maps.

    Configurable properties
    This class has configurable properties that can be set through the BeanContextFactory.setProperty(String, Object) method.

    See BeanContextProperties for settings applicable to this class.

    Examples

    // Construct a context from scratch. BeanContext beanContext = new BeanContextFactory() .setProperty(BeanContextProperties.BEAN_beansRequireDefaultConstructor, true) .addNotBeanClasses(Foo.class) .getBeanContext(); // Clone an existing context factory. BeanContext beanContext = otherBeanContextFactory.clone() .setProperty(BeanContextProperties.BEAN_beansRequireDefaultConstructor, true) .addNotBeanClasses(Foo.class) .getBeanContext();

    Bean Maps

    BeanMaps are wrappers around Java beans that allow properties to be retrieved and set using the common Map.put(Object,Object) and Map.get(Object) methods.

    Bean maps are created in two ways...

    1. BeanContext.forBean() - Wraps an existing bean inside a Map wrapper.
    2. BeanContext.newInstance() - Create a new bean instance wrapped in a Map wrapper.
    Examples

    // A sample bean class public class Person { public String getName(); public void setName(String name); public int getAge(); public void setAge(int age); } // Wrap an existing bean in a new bean map BeanMap<Person> m1 = BeanContext.DEFAULT.forBean(new Person()); m1.put("name", "John Smith"); m1.put("age", 45); // Create a new bean instance wrapped in a new bean map BeanMap<Person> m2 = BeanContext.DEFAULT.newInstance(Person.class); m2.put("name", "John Smith"); m2.put("age", 45); Person p = m2.getBean(); // Get the bean instance that was created.

    Bean Annotations

    This package contains annotations that can be applied to class definitions to override what properties are detected on a bean.

    Examples

    // Bean class definition where only property 'name' is detected. @Bean(properties={"name"}) public class Person { public String getName(); public void setName(String name); public int getAge(); public void setAge(int age); }

    See @Bean and @BeanProperty for more information.

    Beans with read-only properties

    Bean maps can also be defined on top of beans with read-only properties by adding a @BeanConstructor annotation to one of the constructors on the bean class. This will allow read-only properties to be set through constructor arguments.

    When the @BeanConstructor annotation is present, bean instantiation is delayed until the call to BeanMap.getBean(). Until then, bean property values are stored in a local cache until getBean() is called. Because of this additional caching step, parsing into read-only beans tends to be slower and use more memory than parsing into beans with writable properties.

    Attempting to call BeanMap.put(String,Object) on a read-only property after calling BeanMap.getBean() will result in a BeanRuntimeException being thrown. Multiple calls to BeanMap.getBean() will return the same bean instance.

    Beans can be defined with a combination of read-only and read-write properties.

    See @BeanConstructor for more information.

    Filters

    Filters are used to tailor how beans and non-beans are handled.
    There are two subclasses of filters:

    1. BeanFilter - Allows you to tailor handling of bean classes. This class can be considered a programmatic equivalent to the Bean annotation when annotating classes are not possible (e.g. you don't have access to the source).
    2. PojoFilter - Allows you to convert objects to serializable forms.

    See com.ibm.juno.core.filter for more information.

    ClassMetas

    The ClassMeta class is a wrapper around Class object that provides cached information about that class (e.g. whether it's a Map or Collection or bean).

    As a general rule, it's best to reuse bean contexts (and therefore serializers and parsers too) whenever possible since it takes some time to populate the internal ClassMeta object cache. By reusing bean contexts, the class type metadata only needs to be calculated once which significantly improves performance.

    See ClassMeta for more information.

      • toArray

        public Object toArray(ClassMeta<?> type,
             Collection<?> list)
        Converts the contents of the specified list into an array.

        Works on both object and primitive arrays.

        In the case of multi-dimensional arrays, the incoming list must contain elements of type n-1 dimension. i.e. if type is int[][] then list must have entries of type int[].

        Parameters:
        type - The type to convert to. Must be an array type.
        list - The contents to populate the array with.
        Returns:
        A new object or primitive array.
