      • addFilters

        public BeanContextFactory addFilters(Class<?>... classes)
                              throws LockedException
        Add filters to this context.

        There are two category of classes that can be passed in through this method:

        • Subclasses of PojoFilter and BeanFilter.
        • Any other class.

        When IFilter classes are specified, they identify objects that need to be transformed into some other type during serialization (and optionally the reverse during parsing).

        When non-IFilter classes are specified, they are wrapped inside BeanFilters. For example, if you have an interface IFoo and a subclass Foo, and you only want properties defined on IFoo to be visible as bean properties for Foo objects, you can simply pass in IFoo.class to this method.

        The following code shows the order in which filters are applied:

        // F3,F4,F1,F2 beanContext.addFilters(F1.class,F2.class); beanContext.addFilters(F3.class,F4.class);

        Parameters:
        classes - One or more classes to add as filters to this context.
        Returns:
        This object (for method chaining).
        Throws:
        LockedException - If lock() was called on this object.
