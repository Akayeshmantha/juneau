public abstract class ResourceGroup
extends RestServletGroupDefault
    Superclass for all REST resource groups.

    In additional to the functionality of the RestServletGroupDefault group, augments the createVarResolver() method with the following additional variable types:

    • $ARG{...} - Command line arguments.
      Resolves values from Microservice.getArgs().
      Example:

      String firstArg = request.getVarResolver().resolve("$ARG{0}"); // First argument. String namedArg = request.getVarResolver().resolve("$ARG{myarg}"); // Named argument (e.g. "myarg=foo").

    • $MF{...} - Manifest file entries.
      Example:

      String mainClass = request.getVarResolver().resolve("$MF{Main-Class}"); // Main class.

      • createVarResolver

        protected StringVarResolver createVarResolver()
        Description copied from class: RestServlet
        Creates the reusable variable resolver for this servlet.

        Subclasses can override this method to provide their own or augment the existing variable provider.

        • $C{...} - Values from the config file returned by RestServlet.getConfig().
        • $S{...} - System properties.
        • $E{...} - Environment variables.
        • $I{...} - Servlet initialization parameters.

        All variables can provide a 2nd parameter as a default value.

        Example: $S{myBooleanProperty,true}".

        Like all other variables, keys and default values can themselves be arbitrarily nested.

        Example: $S{$E{BOOLEAN_PROPERTY_NAME},$E{BOOLEAN_DEFAULT}}".

        Subclasses can augment this list by adding their own variables.

        Example

        @Override protected StringVarResolver createVarResolver() { StringVarResolver r = super.createVarResolver(); // Wrap all strings inside [] brackets // e.g. "$BRACKET{foobar}" -> "[foobar]" r.addVar("BRACKET", new StringVar() { @Override public String resolve(String varVal) { return '[' + varVal + ']'; } }); return s; }
