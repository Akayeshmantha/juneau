      • onError

        protected void onError(HttpServletRequest req,
           HttpServletResponse res,
           RestException e)
        Callback method for logging errors during HTTP requests.

        Typically, subclasses will override this method and log errors themselves.

        The default implementation simply logs errors to the RestServlet logger.

        Here's a typical implementation showing how stack trace hashing can be used to reduce log file sizes...

        protected void onError(HttpServletRequest req, HttpServletResponse res, RestException e, boolean noTrace) { String qs = req.getQueryString(); String msg = "HTTP " + req.getMethod() + " " + e.getStatus() + " " + req.getRequestURI() + (qs == null ? "" : "?" + qs); int c = e.getOccurrence(); // REST_useStackTraceHashes is disabled, so we have to log the exception every time. if (c == 0) myLogger.log(Level.WARNING, format("[%s] %s", e.getStatus(), msg), e); // This is the first time we've countered this error, so log a stack trace // unless ?noTrace was passed in as a URL parameter. else if (c == 1 && ! noTrace) myLogger.log(Level.WARNING, format("[%h.%s.%s] %s", e.hashCode(), e.getStatus(), c, msg), e); // This error occurred before. // Only log the message, not the stack trace. else myLogger.log(Level.WARNING, format("[%h.%s.%s] %s, %s", e.hashCode(), e.getStatus(), c, msg, e.getLocalizedMessage())); }

        Parameters:
        req - The servlet request object.
        res - The servlet response object.
        e - Exception indicating what error occurred.

      • createVarResolver

        protected StringVarResolver createVarResolver()
        Creates the reusable variable resolver for this servlet.

        Subclasses can override this method to provide their own or augment the existing variable provider.

        • $C{...} - Values from the config file returned by getConfig().
        • $S{...} - System properties.
        • $E{...} - Environment variables.
        • $I{...} - Servlet initialization parameters.

        All variables can provide a 2nd parameter as a default value.

        Example: $S{myBooleanProperty,true}".

        Like all other variables, keys and default values can themselves be arbitrarily nested.

        Example: $S{$E{BOOLEAN_PROPERTY_NAME},$E{BOOLEAN_DEFAULT}}".

        Subclasses can augment this list by adding their own variables.

        Example

        @Override protected StringVarResolver createVarResolver() { StringVarResolver r = super.createVarResolver(); // Wrap all strings inside [] brackets // e.g. "$BRACKET{foobar}" -> "[foobar]" r.addVar("BRACKET", new StringVar() { @Override public String resolve(String varVal) { return '[' + varVal + ']'; } }); return s; }

        Returns:
        The reusable variable resolver for this servlet.

      • createRequestVarResolver

        public StringVarResolver createRequestVarResolver(RestRequest req)
        Returns the variable resolver for the specified request.

        Augments the variable resolver for the servlet getVarResolver() with request-specific variables. Variable resolvers are used to replace variables in property values.

        In addition to the servlet-level variables available on getVarResolver(), adds the following request-specific variables:

        • $L{...} - Localized strings pulled from resource bundle.
          Resolves to the value returned by RestRequest.getMessage(String, Object...).
          Can include message arguments (e.g. "$L{my.localized.string, foo, bar}"
        • $A{...} - Request attributes.
          Resolves to the value returned by ServletRequestWrapper.getAttribute(String).
          Can include a default value (e.g. "$A{myAttribute,myDefaultValue}"
        • $P{...} - Request parameters.
          Resolves to the value returned by RestRequest.getParameter(String).
          Can include a default value (e.g. "$P{myAttribute,myDefaultValue}"
        • $R{...} - Request variables.
          Possible values:
          • $R{contextPath} - Value returned by HttpServletRequestWrapper.getContextPath().
          • $R{method} - Value returned by RestRequest.getMethod().
          • $R{methodDescription} - Value returned by RestRequest.getMethodDescription().
          • $R{pathInfo} - Value returned by HttpServletRequestWrapper.getPathInfo().
          • $R{requestParentURI} - Value returned by RestRequest.getRequestParentURI().
          • $R{requestURI} - Value returned by HttpServletRequestWrapper.getRequestURI().
          • $R{servletDescription} - Value returned by RestRequest.getServletDescription().
          • $R{servletLabel} - Value returned by RestRequest.getServletLabel().
          • $R{servletParentURI} - Value returned by RestRequest.getServletParentURI().
          • $R{servletPath} - Value returned by HttpServletRequestWrapper.getServletPath().
          • $R{servletURI} - Value returned by RestRequest.getServletURI().
          • $R{trimmedRequestURI} - Value returned by RestRequest.getTrimmedRequestURI().
        • $UE{...} - URL-Encode the specified value.
          Takes the contents inside the varible and replaces it with the value returned by calling RestUtils.encode(String).

        Subclasses can override this method to provide their own request-specific replacement variables. See createVarResolver() for an example.

        Parameters:
        req - The HTTP request.
        Returns:
        The string var resolver to use for this request.
