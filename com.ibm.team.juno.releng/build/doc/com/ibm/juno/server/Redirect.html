com.ibm.juno.server

Class Redirect

  • java.lang.Object
    • com.ibm.juno.server.Redirect

  • public final class Redirect
extends Object
    REST methods can return this object as a shortcut for performing HTTP 302 redirects.

    The following example shows the difference between handling redirects via the RestRequest/RestResponse, and the simplified approach of using this class.

    // Redirect to "/contextPath/servletPath/foobar" // Using RestRequest and RestResponse @RestMethod(name="GET", path="/example1") public void example1(RestRequest req, RestResponse res) throws IOException { res.sendRedirect(req.getServletURI() + "/foobar"); } // Using Redirect @RestMethod(name="GET", path="/example2") public Redirect example2() { return new Redirect("foobar"); }

    The constructor can use a MessageFormat-style pattern with multiple arguments:

    @RestMethod(name="GET", path="/example3") public Redirect example3() { return new Redirect("foo/{0}/bar/{1}", id1, id2); }

    The arguments are serialized to strings using the servlet's UrlEncodingSerializer, so any filters defined on the serializer or REST method/class will be used when present. The arguments will also be automatically URL-encoded.

    Redirecting to the servlet root can be accomplished by simply using the no-arg constructor.

    // Simply redirect to the servlet root. // Equivalent to res.sendRedirect(req.getServletURI()). @RestMethod(name="GET", path="/example4") public Redirect exmaple4() { return new Redirect(); }

    This class is handled by RedirectHandler, a built-in default response handler created by RestServlet.createResponseHandlers().
